# Documentation & Thesis Expert Skill

Specialized guidance for creating high-impact documentation and structured academic theses.

## Technical Documentation & READMEs

### The Professional README Structure
1.  **Tagline & Hero:** Clear value proposition and a visual demo (GIF/SVG).
2.  **Quick Start:** The "under 2 minutes" path to a working system.
3.  **Architecture:** High-level block diagrams explaining the "How" and "Why".
4.  **Deployment/Installation:** Copy-pasteable commands with prerequisite checks.
5.  **Roadmap & Contribution:** Transparency on current status and how others can help.

### Documentation Principles (The Four Pillars)
- **Tutorials:** Learning-oriented, step-by-step for beginners.
- **How-to Guides:** Goal-oriented, solving specific problems for users.
- **Reference:** Information-oriented, technical specs for developers.
- **Explanation:** Understanding-oriented, deep dives into design decisions.

## Academic & Thesis Writing

### Thesis Structure (Standard STEM)
- **Abstract:** Concise summary of problem, method, and key findings.
- **Introduction:** Context, motivation, and explicit research questions.
- **State of the Art (Related Work):** Positioning against existing solutions (e.g., SayCan, VoxPoser for Go2).
- **Methodology:** Detailed description of the system (DDS, FAST-LIO2, MCP).
- **Experimental Setup & Results:** Quantitative data (RMSE, latency) and qualitative analysis.
- **Discussion & Conclusion:** Implications of the work and future directions.

### Tools & Formatting
- **LaTeX:** Use for complex bibliographies, cross-referencing, and perfect typesetting.
- **Markdown/Quarto:** Use for rapid drafting and documentation that needs to live alongside code.
- **Zotero/BibTeX:** Maintain a rigorous citation database from day one.

## Tone & Style
- **Clarity over Jargon:** Explain acronyms on first use (e.g., "Model Context Protocol (MCP)").
- **Active Voice:** "The system processes the data" rather than "Data is processed by the system".
- **Visual Consistency:** Use consistent styling for diagrams (e.g., Mermaid.js or Lucidchart).
